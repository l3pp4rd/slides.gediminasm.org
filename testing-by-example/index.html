Testing by example

In other words - I wanted to share some practical cases on how I approach TDD in projects I work with.

My point is

Whomever higher in the hierarchy:

We want this feature done ASP!

Yeah right.. doesn't it sound like?

OR

First thing, lets do this, we will write tests later..

Yeah right.. we know how it ends, don't we?

Where to start?

  • Define features - BDD Behat
  • Make a proof of concept for components - design the behavior
  • Unit test the components of feature - Phpunit phpspec
  • Loop around until it is solid and most important - until you are proud of it

Well, and write the actual code at some point.

Feature

Feature: EgoPay deposits
  In order to have some money in my balance
  As a confirmed user
  I need to be able to deposit with EgoPay

  Background:
    Given the site has the following users:
      | username       | password | role           |
      | gedi@gmail.com | secret   | ROLE_CONFIRMED |
    And I'm logged in as "gedi@gmail.com" with password "secret"

  Scenario: Deposit with EgoPay
    Given I am on page "EgoPay deposit"
    When I fill in "Amount" with "200"
    And I press "Deposit"
    Then I should see "$200.00" amount to confirm on EgoPay page
    When I press "Next"
    And I press "Confirm"
    Then I should see a success notification "You have deposited $200.00 into your wallet"

Design the behavior

<?php // imaginable depositAction in controller

$transfer = new Transfer

$form = $this->createForm(new TransferType, $transfer);
$form->handleRequest($request);

// bind amount and currency - request to form, validate
if ($form->isValid()) {
    $transfer->setStatus(Transfer::PENDING);
    // link to an user and his wallet
    // persist $transfer and redirect to egopay
    $this->get('egopay')->create($transfer);
}
