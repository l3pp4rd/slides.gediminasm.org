What is ViM?

“Vim is a highly configurable text editor built to enable efficient text editing. It is an improved version of the vi editor distributed with most UNIX systems.”

What Vim Is Not?

“Vim isn't an editor designed to hold its users' hands. It is a tool, the use of which must be learned.”

Why I should use ViM?

Mainly, because you would join the geek force!

That was not so convincing, was it?

  • Because it is efficient, extensible and fun!
  • Pair programming with tmux
  • Any programming language support
  • Modal editing - insert, visual, command modes
  • Vim is free and has huge community and it was there for decades, while other editors were forgotten or changed
  • You can hack Vim to behave like IDE and in any other way you wish
  • You can run hundreds of ViM instances for separate projects with minimal OS resources
  • You can have the same vim configuration on separate machines, no difference through ssh

How can it replace my favourite IDE?

Autocompletion

Supported through plugins, ctags. It will not be as feature rich as in IDE out of the box. But it can be configured to be. Here is a list of common plugins for autocompletion:

  • Supertab - use Tab for autocompletion
  • php completion - autocompletion plugin for php. Can switch to this plugin based on file extension currently edited.
  • YCM a complete engine for autocompletion

Generally speaking, autocompletion can be as feature rich as in IDE. Depends on your needs, how far you want it to be
